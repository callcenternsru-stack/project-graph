1. Введение

Проект представляет собой комплексную экосистему для управления колл-центром. Он включает модули учёта рабочего времени (с синхронизацией через Netlify Blobs) и рекрутинга (с хранением данных в localStorage). В последних версиях добавлена поддержка анкетирования кандидатов и программа автоматической проверки.

2. Архитектура системы

Система построена на статических HTML-страницах, JavaScript (клиентская логика) и Netlify Functions для работы с хранилищем. Данные учёта времени хранятся в Netlify Blobs, синхронизация происходит через функции. Данные рекрутинга (рекрутеры, кандидаты, скрипты, анкеты) хранятся локально в localStorage и не синхронизируются с сервером. Аутентификация для партнёров реализована через сессии (sessionStorage), для рекрутеров вход не требуется (используется первый рекрутер из списка).

3. Настройка окружения

Для разработки и локального тестирования необходим любой современный браузер. Для работы с Netlify Blobs требуется аккаунт Netlify и настроенные переменные окружения. Подробные инструкции по деплою приведены в разделе 9.

5. Формат данных в Blobs

Используется одно хранилище app-data, в котором сохраняется полное состояние приложения учёта времени: проекты, месяцы, сотрудники, записи времени, партнёры, комментарии. Данные рекрутинга в Blobs не сохраняются.

6. Серверные функции (Netlify Functions)

6.1 getData – загрузка данных

Возвращает текущее состояние из хранилища app-data. Используется страницами index.html, partner.html, view.html для инициализации.

6.2 saveData – сохранение данных

Принимает POST-запрос с полным состоянием и сохраняет его в app-data. Вызывается при любых изменениях в модуле учёта времени.

6.3 submitResult – (зарезервировано, не используется)

Функция предназначена для приёма результатов от программы автоматической проверки, но в текущей версии не задействована.

6.4 getFile – (зарезервировано, не используется)

Функция для выдачи файлов результатов, в настоящее время не используется.

7. Клиентские страницы

7.1 index.html – админ-панель учёта времени

Полнофункциональная панель для администратора: управление проектами, сотрудниками, партнёрами, просмотр графиков и статистики. Поддерживает синхронизацию с Netlify Blobs через функции getData и saveData.

7.2 partner.html – панель партнёра

Аналог админ-панели, но с ограниченным доступом – партнёр видит только свои проекты и своих сотрудников. Вход по логину/паролю из массива partners, хранящегося в Blobs. Данные синхронизируются с сервером.

7.3 view.html – публичный просмотр

Страница только для чтения, позволяет любому пользователю выбрать проект и месяц и увидеть график работы. Данные загружаются через getData.

7.4 recruiter.html – панель рекрутера (локальная)

Позволяет рекрутеру просматривать скрипты и обрабатывать своих кандидатов. Все данные хранятся в localStorage (ключ recruiting_local_data). Синхронизация с сервером отсутствует. При входе автоматически выбирается первый рекрутер из списка.

7.5 hr control panel.html – админ-панель рекрутинга (локальная)

Основная панель для управления рекрутингом. Включает вкладки:

  • «Рекрутеры» – управление учётными записями рекрутеров.
  • «База контактов» – список кандидатов, добавленных вручную или импортированных из Excel.
  • «Скрипты» – редактирование текстов скриптов для рекрутеров.
  • «График» – планирование организационных встреч и обучений (данные хранятся локально).
  • «Анкеты авто» – отображение анкет, заполненных на странице candidate.html при выборе автоматической проверки (сохраняются в localStorage).
  • «Анкеты ручные» – отображение анкет, заполненных при выборе ручной проверки.

Все данные рекрутинга хранятся исключительно в localStorage и не синхронизируются с Netlify.

7.6 candidate.html – анкета кандидата (проект РОССЕТТИ, локальная)

Страница для сбора первичных данных кандидата. Содержит:

  • Информацию о проекте, требованиях, FAQ.
  • Форму с обязательными полями: ФИО, русский псевдоним, Telegram, телефон, email.
  • Две кнопки: «Пройти проверку автоматически» и «Пройти проверку вручную».
  • При нажатии на авто-проверку генерируется уникальный код (формата ROSSETI-XXXX-XXXX), данные формы сохраняются в массив autoForms в localStorage, показывается блок с кодом и инструкцией.
  • При нажатии на ручную проверку данные сохраняются в массив manualForms и показывается инструкция по отправке скриншотов.
  • Реализована защита от дублирования: если кандидат с такими же данными уже существует в соответствующем массиве, запись заменяется (обновляется код и timestamp).

Данные анкет не отправляются на сервер, хранятся локально в браузере.

8. Аутентификация и сессии

Для партнёров используется проверка логина/пароля из массива partners, хранящегося в Netlify Blobs. Сессия сохраняется в sessionStorage. Для рекрутеров вход не требуется – панель рекрутера автоматически использует первого рекрутера из локального хранилища.

9. Инструкция по развёртыванию на Netlify

Проект готов к деплою на Netlify. Необходимые шаги:

  1. Собрать все файлы в структуру, описанную в разделе 4.
  2. Убедиться, что в package.json указана зависимость @netlify/blobs.
  3. Скомпилировать SystemCheckPRO.ps1 в EXE и поместить в папку public.
  4. Загрузить репозиторий на GitHub и подключить к Netlify.
  5. Указать в настройках Netlify папку публикации public и папку функций functions.
  6. Добавить переменные окружения NETLIFY_SITE_ID и NETLIFY_ACCESS_TOKEN (токен из UI Netlify).
  7. Выполнить деплой.

После деплоя функции getData и saveData будут доступны, обеспечивая синхронизацию модуля учёта времени. Страницы рекрутинга продолжат работать локально (данные остаются в браузере пользователя).

10. Локальное тестирование (без сервера)

Для локального тестирования достаточно открыть HTML-файлы в браузере. Данные учёта времени будут сохраняться в localStorage (но без синхронизации с Netlify). Для полноценной проверки функций, связанных с Blobs, рекомендуется использовать Netlify CLI или локальный сервер с эмуляцией функций.

11. Возможные проблемы

  • Данные рекрутинга не видны в разных браузерах – используются разные хранилища localStorage, общих данных нет.
  • Программа SystemCheckPRO не отправляет результаты на сервер – в текущей версии программа сохраняет файлы локально, отправка не реализована.
  • Конфликт версий localStorage – при изменении структуры данных старые записи могут вызывать ошибки; рекомендуется очищать localStorage через инструменты разработчика.

13. Программа автоматической проверки кандидата (SystemCheckPRO.exe)

13.1. Назначение и возможности

Автономная программа для Windows, которая позволяет кандидату предоставить информацию о компьютере, пройти тест скорости печати и записать голосовой образец. Программа не требует установки, распространяется в виде одного .exe файла с дополнительными утилитами.

13.2. Собираемые данные

  • Характеристики ПК: процессор, ядра, ОЗУ, тип и объём диска, ОС.
  • Тип подключения к интернету (Wi-Fi / Ethernet).
  • Детальный анализ аудиоустройств (USB-гарнитуры, Bluetooth и т.д.).
  • Тест скорости печати: скорость (знаков/мин), ошибки, точность.
  • Запись голоса через микрофон (с помощью утилиты SoX).
  • Скорость интернета (пинг, загрузка, отдача) с помощью Speedtest CLI.

13.3. Использование и распространение

Программа распространяется в ZIP-архиве вместе с sox.exe и необходимыми DLL. Кандидат запускает SystemCheckPRO.exe, вводит уникальный код, последовательно проходит тесты. Результаты сохраняются локально в папке с программой в файлах results.json и report.txt.

13.4. Форматы отчётов

  • results.json – расширенный отчёт в формате JSON.
  • report.txt – упрощённый текстовый отчёт для пользователя.

13.5. Ручная проверка (альтернативный вариант)

Кандидат может предоставить скриншоты необходимых экранов (диспетчер задач, свойства системы, тест скорости печати) и голосовую запись, отправив их на почту. Этот вариант используется, если запуск .exe невозможен.

13.6. Последние улучшения (версия 5.1)

  • Приветственное окно в корпоративном стиле.
  • Окно загрузки во время сбора характеристик.
  • Тест печати с подсветкой совпадающих символов, запрет навигации.
  • Автоматическое завершение теста при достижении конца текста.
  • Кнопка «НАЧАТЬ ЗАНОВО» для сброса теста.

13.7. Дальнейшие улучшения (версия 5.2)

  • Три различных текста для теста печати.
  • Кнопка «ПЕРЕЙТИ К СЛЕДУЮЩЕМУ ЗАДАНИЮ» появляется после завершения теста.
  • В окне записи голоса – реалистичный сценарий разговора с техподдержкой, мигающая красная надпись во время записи.
  • Интеграция со Speedtest CLI (тесты по трём городам).
  • Сбор характеристик ПК и анализ аудиоустройств.

13.8. Компиляция и распространение

Скрипт компилируется в .exe с помощью модуля ps2exe. Вместе с EXE-файлом необходимо распространять sox.exe. При первом запуске программа автоматически загружает Speedtest CLI (если отсутствует).

13.9. Заключение

Программа SystemCheckPRO.exe представляет собой законченное решение для автоматизированной проверки кандидатов, включающее тестирование скорости печати, запись голоса, анализ характеристик ПК и измерение скорости интернета.

13.10. Интеграция с веб-частью (текущее состояние)

В текущей версии интеграция программы с веб-интерфейсом ограничена:

  • На странице candidate.html генерируется уникальный код, который кандидат должен ввести в программу. Программа сохраняет этот код в отчёте.
  • После завершения тестов программа сохраняет результаты локально. Отправка результатов на сервер (Netlify) не реализована.
  • В HR-панели во вкладках «Анкеты авто» и «Анкеты ручные» отображаются только данные анкет (ФИО, контакты, код). Статус проверки, ссылки на отчёты и аудиоплеер отсутствуют, так как результаты не загружаются на сервер.

Планируется доработка: добавление функций submitResult и getFile для приёма результатов и их отображения в HR-панели, но на данный момент эти функции не используются.
