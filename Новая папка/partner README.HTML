2. Структура и семантика

Страница разделена на логические блоки, оформленные с помощью семантических и пользовательских элементов.

  • Боковое меню (выезжающее) – .side-menu-trigger + .side-menu с кнопками: Статистика, Фильтр, Сотрудники, График, Скрыть поля, Выйти.
  • Форма входа (.login-container) – появляется первой, содержит поля логин/пароль и кнопку «Войти».
  • Основной интерфейс партнёра (.partner-container) – скрыт до успешного входа.
  • Шапка партнёра (.partner-header) – отображает название партнёра, текущий месяц и индикатор синхронизации.
  • Вкладки проектов (.project-tabs) – динамически формируются из проектов, доступных партнёру.
  • Основная область контента (#mainContent) – сюда рендерится либо страница графика (.schedule-wrapper), либо страница статистики (.stats-page).
  • Инструкция по цветам (.instruction) – пояснение цветовых маркеров для статусов ячеек.
  • Контекстные меню (модальные окна) – .context-menu для фильтра, сотрудников, графика, комментариев и сводной таблицы. Появляются по клику на соответствующие пункты бокового меню.

Вёрстка адаптивная: используются grid для вкладок и карточек статистики, flex для шапки, медиазапросы (неявно через grid-template-columns с auto-fit).

3. Стилизация (CSS)

Корпоративная тёмная тема: фон #000 и #111, акцентный цвет #ffd700 (золотой/жёлтый), текст белый/серый.

  • Боковое меню – выезжает при наведении на левый край экрана (.side-menu-trigger). Пункты меню с жёлтым текстом, при наведении появляется левая жёлтая граница.
  • Форма входа – тёмная карточка с жёлтой рамкой, поля ввода с фокусом жёлтые.
  • Вкладки проектов – сетка из 10 колонок, активная вкладка с жёлтой нижней границей.
  • График – сложная табличная структура с фиксированными левыми колонками и горизонтальным скроллом. Ячейки времени (.time-cell input) окрашиваются в зависимости от статуса (опоздание, переработка, отпуск и т.д.) – для этого используются классы .late, .overtime, .vacation, .sick, .absent, .dayoff.
  • Индикатор комментария – маленький жёлтый кружок в ячейке, если есть комментарий.
  • Контекстные меню – центрированные модальные окна с анимацией появления, с формой внутри.
  • Статистика – карточки (.stat-card) с крупными цифрами, при клике на некоторые открывается сводная таблица. Таблица статистики с сортировкой по клику на заголовок.
  • Скрытие полей – кнопка в меню переключает класс .hidden для дополнительных колонок (смена, время, телефон, тип).

Все стили находятся внутри тега <style> и не используют внешние библиотеки.

4. JavaScript (логика и интерактивность)

Код написан на чистом JavaScript (ES6) и включает следующие ключевые функции и модули:

  • Работа с хранилищем:
    • loadFromSheets() – загружает данные из Netlify Blobs через getData и сохраняет в localStorage (ключи STORAGE_KEYS).
    • saveToSheets() – отправляет текущее состояние (из localStorage) на сервер через saveData.
    • loadData() – читает данные из localStorage.
    • showSync() / showSyncError() – меняют цвет индикатора синхронизации.
  • Аутентификация и сессия:
    • partnerLogin() – проверяет логин/пароль по массиву partners из loadData(). При успехе сохраняет сессию в sessionStorage и инициализирует интерфейс.
    • logout() – очищает сессию и возвращает к форме входа.
    • loadSession() – восстанавливает сессию при загрузке страницы.
  • Управление меню:
    • toggleMenu(menuId), closeMenu(), closeAllMenus() – открывают/закрывают контекстные меню.
  • Работа с проектами и месяцами:
    • switchProject() – переключает текущий проект, обновляет месяц и UI.
    • addMonth(), deleteMonth() – добавление/удаление месяца для проекта.
    • updateMonthsList() – обновляет список месяцев в меню «График».
  • Управление сотрудниками:
    • addEmployee() – добавляет оператора (имя + (партнёр)) в выбранный проект/месяц.
    • deleteEmployee() – удаляет оператора и все связанные записи.
    • updateEmployeeData() – обновляет поля смены, времени, телефона сотрудника.
    • updateTimeEntry() – сохраняет значение времени в конкретной ячейке дня, при необходимости вызывает модалку комментария.
    • getPartnerEmployees() – фильтрует сотрудников, принадлежащих текущему партнёру.
  • Комментарии:
    • forceCommentModal() – принудительно открывает окно комментария при опоздании/переработке/прогуле.
    • openCommentModal(), saveComment(), deleteComment() – работа с комментариями.
    • getCommentKey() – формирует уникальный ключ для комментария.
  • Отображение страниц:
    • renderSchedulePage() – строит таблицу графика с учётом сортировки, скрытых колонок, статусов и комментариев.
    • renderStatsPage() – строит страницу статистики с карточками и таблицей, поддерживает сортировку.
    • refreshUI() – обновляет все элементы интерфейса после изменений.
  • Сортировка и фильтры:
    • sortSchedule(), sortStats() – сортировка в графике и статистике.
    • applyFilter() – применяет выбранный проект и месяц из меню «Фильтр».
  • Вспомогательные функции:
    • formatMonth(), getDaysInMonth(), getDayOfWeek(), validateTimeRange(), getDurationHours(), getCellStatus() – работа с датами и временем.
    • exportToExcel() – выгружает статистику в CSV.
    • showSummaryTable(), collectSummaryData(), renderSummaryTable() – сводные таблицы по опозданиям/переработкам/прогулам.

5. Формы и отправка данных

На странице нет единой формы отправки, но есть несколько интерактивных элементов, которые инициируют сохранение:

  • Вход – форма с полями логин/пароль, отправка через partnerLogin() (проверка по данным из localStorage).
  • Добавление сотрудника – поле ввода ФИО + кнопка «Добавить оператора»; данные сохраняются через addEmployee() и затем saveToSheets().
  • Редактирование полей сотрудника – <select> и <input> в фиксированных колонках вызывают updateEmployeeData() при изменении.
  • Редактирование ячеек времени – <input> в каждой ячейке дня вызывают updateTimeEntry() при потере фокуса (onchange).
  • Добавление месяца – поле input type="month" + кнопка в меню «График».
  • Сохранение комментария – форма в модальном окне с полем textarea и кнопками «Сохранить» / «Удалить».

Все изменения немедленно сохраняются в localStorage и, если возможно, отправляются на сервер через saveToSheets() (асинхронно).

6. Особенности реализации

  • Двойное хранение данных: localStorage используется как основной кэш, сервер (Netlify Blobs) – как постоянное хранилище. При загрузке страницы данные сначала загружаются с сервера и помещаются в localStorage. При любых изменениях вызывается saveToSheets(), который отправляет полный слепок данных на сервер.
  • Идентификация сотрудников партнёра: к имени сотрудника добавляется суффикс (Имя партнёра), а в поле partner объекта employeeData сохраняется имя партнёра. Это позволяет фильтровать сотрудников, принадлежащих только данному партнёру.
  • Принудительный комментарий: если пользователь вводит в ячейку значение, которое интерпретируется как опоздание, переработка или прогул, автоматически открывается модальное окно с требованием указать причину (поле обязательно). Без сохранения комментария изменение не фиксируется.
  • Скрытие/показ колонок: реализовано через класс .hidden и пересчёт ширины фиксированных контейнеров (fixedCells.style.width).
  • Сортировка в графике: сохраняется в localStorage, чтобы при перезагрузке страницы сортировка не сбрасывалась.
  • Индикатор синхронизации: зелёный пульсирующий кружок при успешной загрузке/сохранении, красный при ошибке.
  • Контекстные меню: реализованы как фиксированные модальные окна, которые показываются по клику на соответствующий пункт бокового меню. При открытии одного остальные закрываются.
  • Валидация времени: функция validateTimeRange() проверяет формат ЧЧ:ММ-ЧЧ:ММ. При вводе неправильного формата показывается сообщение, и сохранение не происходит.
  • Автоматическое заполнение ячеек: при установке базового времени сотрудника (employeeData.time) пустые ячейки всех дней автоматически заполняются этим значением (для удобства).

7. Потенциальные точки расширения

  • Управление партнёрами через админку: сейчас партнёры хранятся в partners внутри app-data. Можно добавить интерфейс для создания/редактирования партнёров в админ-панели (index.html).
  • Ролевая модель: расширить права партнёров (например, разрешить только просмотр, без редактирования).
  • Уведомления об изменениях: добавить WebSocket или long polling для уведомления партнёров о внесённых изменениях другими пользователями.
  • Экспорт графика в Excel: помимо статистики, можно добавить выгрузку всего графика.
  • Фильтр по сотрудникам: в графике можно добавить поиск или фильтрацию по имени.
  • История изменений: хранить лог изменений (кто и когда менял ячейки) для аудита.
  • Поддержка мобильных устройств: улучшить адаптивность для маленьких экранов (возможно, упростить таблицу).
  • Интеграция с календарями: позволить экспортировать график в iCal.

8. Особенности для тестирования

  • Локальное тестирование без сервера: при открытии file:// запросы к /.netlify/functions/... будут неудачными. Для тестирования рекомендуется использовать Netlify CLI или развернуть сайт на Netlify.
  • Тестовые учётные данные: в localStorage после первой загрузки с сервера должны появиться партнёры (например, из демо-данных). Можно создать партнёра через админ-панель.
  • Проверка синхронизации: после внесения изменений зелёный индикатор должен мигнуть. При отключении интернета или ошибке сервера индикатор станет красным.
  • Добавление сотрудника: в меню «Сотрудники» выберите проект и месяц, введите имя и нажмите «Добавить». Сотрудник должен появиться в графике.
  • Редактирование ячеек: введите в ячейку время (например, 09:00-18:00). Если оно меньше базового времени, ячейка должна подсветиться жёлтым (.late) и появиться модалка комментария.
  • Комментарии: попробуйте сохранить комментарий – в ячейке появится жёлтый кружок. Клик на него снова откроет комментарий.
  • Скрытие колонок: нажмите в боковом меню «Скрыть поля» – дополнительные колонки должны исчезнуть, ширина фиксированной части изменится.
  • Статистика: переключитесь на страницу статистики. Карточки должны показывать суммы. Клик на карточки «Опоздания», «Переработки», «Прогулы» должен открывать сводную таблицу.
  • Экспорт в Excel: на странице статистики нажмите «📥 Excel» – должен скачаться CSV-файл с данными.
  • Добавление месяца: в меню «График» выберите проект и введите месяц, нажмите «Добавить». Месяц должен появиться в списке и стать текущим.
  • Удаление месяца: в том же меню нажмите крестик рядом с месяцем – после подтверждения месяц и все связанные данные должны удалиться.
