📄 Инструкция: recruiter.html

Панель рекрутера для управления кандидатами и просмотра скриптов (модуль рекрутинга).

Файл расположен: C:\Users\Контакт-Сервис\Desktop\prodect\public\recruiter.html

2. Структура и семантика

Страница разделена на логические блоки, оформленные с помощью семантических и пользовательских элементов.

  • Боковое меню (выезжающее) – .side-menu-trigger + .side-menu с кнопками: Главная, Выйти.
  • Форма входа (.login-container) – появляется первой, содержит поля логин/пароль и кнопку «Войти».
  • Основная панель рекрутера (.recruiter-interface) – скрыта до успешного входа.
  • Шапка (.header) – отображает имя рекрутера и кнопки: «График» (переход в режим просмотра) и «Выйти».
  • Левая панель (скрипты) – .left-panel содержит выпадающий список скриптов (#scriptSelector) и контейнер для отображения выбранного скрипта (.script-item).
  • Правая панель (форма кандидата) – .right-panel содержит форму (#candidateForm) с полями: ФИО, телефон, проект, страна, дата обучения, время обучения, результат звонка, тип обращения, комментарий. Также присутствует счётчик кандидатов и скрытые поля.
  • Уведомления – динамически создаваемые блоки .notification для всплывающих сообщений.

Вёрстка адаптивная: используется flex для расположения панелей, grid для полей формы, медиазапросы (неявно).

3. Стилизация (CSS)

Корпоративная тёмная тема: фон #000 и #111, акцентный цвет #ffd700 (золотой/жёлтый), текст белый/серый.

  • Боковое меню – выезжает при наведении на левый край экрана (.side-menu-trigger). Пункты меню с жёлтым текстом, при наведении появляется левая жёлтая граница. Кнопка выхода красная.
  • Форма входа – тёмная карточка с жёлтой рамкой, поля ввода с фокусом жёлтые.
  • Шапка – тёмный фон, жёлтый заголовок, кнопки синяя (график) и красная (выход).
  • Левая панель – селект скриптов в тёмном стиле, блок скрипта с жёлтой левой границей и прокруткой содержимого.
  • Правая панель – форма с полями в тёмном стиле, обязательные поля помечены красной звёздочкой. Кнопка сохранения жёлтая, при наведении поднимается.
  • Уведомления – фиксированные в правом нижнем углу, с цветной рамкой (зелёная – успех, красная – ошибка).
  • Счётчик кандидатов – серый текст справа над формой.

4. JavaScript (логика и интерактивность)

Код написан на чистом JavaScript (ES6) и включает следующие ключевые функции и модули:

  • Глобальные переменные:
    • currentRecruiter – объект с данными текущего рекрутера.
    • myCandidates – массив кандидатов, закреплённых за рекрутером.
    • currentCandidateIndex – индекс текущего отображаемого кандидата.
    • currentScriptTitle – название выбранного скрипта.
    • appData – объект со справочными данными (проекты, результаты звонков, типы обращений, скрипты, даты обучений).
  • Уведомления – showNotification(message, type) создаёт и показывает всплывающее сообщение.
  • Аутентификация:
    • login() – отправляет POST-запрос на /.netlify/functions/loginRecruiter с логином и паролем. При успехе сохраняет рекрутера в sessionStorage и инициализирует интерфейс.
    • showLoginError() – отображает ошибку входа.
    • logout() – удаляет сессию и перезагружает страницу.
    • showForm() – прокручивает страницу вверх.
  • Инициализация:
    • initInterface() – после входа загружает справочники, скрипты и кандидатов, затем отображает первого кандидата (если есть) или пустую форму.
    • loadReferenceData() – параллельно загружает проекты, результаты звонков, типы обращений и даты обучений через соответствующие функции.
    • loadScripts() – загружает список скриптов через getScripts и обновляет UI.
    • loadMyCandidates() – загружает кандидатов текущего рекрутера через getMyCandidates?recruiter=..., сортирует по дате создания (новые сверху).
  • Заполнение выпадающих списков:
    • fillProjects(), fillCallResults(), fillContactTypes() – заполняют соответствующие <select> данными из appData.
  • Работа со скриптами:
    • updateScriptsUI() – обновляет селект скриптов, пытается восстановить ранее выбранный скрипт, отображает его содержимое.
    • displayScript(index) – показывает заголовок и HTML-содержимое выбранного скрипта.
    • setProjectFromScript(scriptTitle) – если проект с таким названием отсутствует в списке, добавляет его; устанавливает проект в форме и обновляет даты обучения.
    • Обработчик #scriptSelector – при смене скрипта обновляет отображение и проект.
  • Даты обучения:
    • updateTrainingDates(project) – извлекает из appData.projectTrainings все даты для указанного проекта (ключи вида проект_ГГГГ-ММ_ДД), сортирует и заполняет селект #trainingDateSelect.
    • onTrainingDateChange() – при выборе даты автоматически подставляет время из атрибута data-time.
    • Обработчик #project – при смене проекта обновляет даты обучения.
  • Загрузка кандидата в форму:
    • loadCandidateIntoForm(idx) – заполняет поля формы данными кандидата с указанным индексом. Если индекс -1, очищает форму (проект устанавливается из скрипта, если он выбран). Также определяет страну по телефону (по коду: +996 → Кыргызстан, +998 → Узбекистан, +77 → Казахстан, +79 → Россия). Обновляет счётчик.
  • Сохранение кандидата:
    • Обработчик submit формы – валидирует обязательные поля, собирает обновлённые данные из формы, отправляет PUT/POST на /.netlify/functions/submitCandidate. При успехе – уведомление и вызов goToNextCandidate().
    • goToNextCandidate() – переходит к следующему кандидату по кругу (если список не пуст) и загружает его в форму.
  • Восстановление сессии – при загрузке страницы проверяет sessionStorage, если там есть рекрутер – автоматически входит.

5. Форма и отправка данных

Форма (#candidateForm) содержит следующие поля:

  • fullName (обязательное) – текстовое поле.
  • phoneNumber (обязательное) – телефон.
  • project (обязательное) – выпадающий список проектов.
  • country – выпадающий список стран (Кыргызстан, Узбекистан, Россия, Казахстан).
  • trainingDateSelect – выбор даты обучения (заполняется динамически).
  • trainingTimeInput – время обучения (текстовое поле, формат чч:мм).
  • callResult – результат звонка (из справочника).
  • contactType – тип обращения (из справочника).
  • comment – многострочный комментарий.
  • Скрытые поля: recruiterField (имя рекрутера), candidateIndexField (индекс текущего кандидата).

Валидация на стороне клиента: проверяется заполненность обязательных полей (ФИО, телефон, проект). Если какое-то поле пустое, показывается уведомление об ошибке, отправка не происходит.

Отправка: POST-запрос на /.netlify/functions/submitCandidate с телом JSON, содержащим весь объект кандидата (включая скрытые поля и обновлённые значения). Сервер обновляет существующую запись (по ключу id).

6. Особенности реализации

  • Отсутствие локального хранения: все данные (кандидаты, справочники) загружаются с сервера при каждом входе или обновлении страницы. Это гарантирует актуальность, но требует постоянного соединения.
  • Автоматическое определение страны: при загрузке кандидата в форму страна устанавливается на основе кода телефона (первые цифры). Это удобно для быстрого заполнения, но может ошибаться, если номер не соответствует формату.
  • Связь скриптов и проектов: название скрипта автоматически становится проектом. Если проект ещё не существует в справочнике, он добавляется на лету (только в локальный массив appData.projects, но не сохраняется на сервере). Это может привести к рассинхронизации.
  • Даты обучения: хранятся в отдельном хранилище projectTrainings в виде плоского объекта с ключами вида проект_ГГГГ-ММ_ДД и значениями времени. Это позволяет гибко добавлять даты для каждого проекта.
  • Переход по кандидатам: после сохранения текущего кандидата происходит переход к следующему по списку (циклически). Это ускоряет обработку последовательных кандидатов.
  • Безопасность: пароли рекрутеров проверяются на сервере, в ответе никогда не передаётся пароль (удаляется). Сессия хранится в sessionStorage и очищается при выходе.

7. Потенциальные точки расширения

  • Создание нового кандидата: сейчас форма позволяет только редактировать существующих кандидатов. Можно добавить кнопку «Новый кандидат», которая очищает форму и создаёт запись с новым id.
  • Фильтрация и поиск кандидатов: можно добавить поле поиска по имени или телефону, а также фильтры по статусу/результату.
  • Валидация форматов: добавить проверку телефона регулярным выражением, формата времени, email (если появится).
  • Загрузка файлов: возможность прикреплять к кандидату файлы (резюме, скриншоты) через multipart/form-data.
  • История изменений: логировать кто и когда менял кандидата.
  • Уведомления о новых кандидатах: периодически опрашивать сервер на наличие новых кандидатов для рекрутера.
  • Групповые операции: например, отметить несколько кандидатов и изменить результат звонка массово.
  • Интеграция с календарём: экспорт дат обучения в календарь.

8. Особенности для тестирования

  • Необходимость сервера: для полноценного тестирования требуется запущенный экземпляр Netlify Functions (локально через Netlify CLI или развёрнутый на Netlify).
  • Тестовые учётные данные: в админ-панели (hr control panel.html) можно создать рекрутера с логином и паролем. Используйте эти данные для входа.
  • Проверка входа: введите неверный логин/пароль – должно появиться сообщение об ошибке. Верные данные должны скрыть форму входа и показать панель.
  • Загрузка данных: после входа убедитесь, что загрузились проекты, скрипты и кандидаты (если есть). В консоли браузера не должно быть ошибок 404 или 500.
  • Выбор скрипта: выберите скрипт из списка – его содержимое должно отобразиться, а проект в форме должен смениться на название скрипта. Если для этого проекта есть даты обучения, они появятся в соответствующем селекте.
  • Редактирование кандидата: измените любое поле и нажмите «Сохранить». Должно появиться зелёное уведомление, и форма переключится на следующего кандидата (если есть). Проверьте в админ-панели, что изменения применились.
  • Автоопределение страны: при загрузке кандидата с номером +996 XXX страница должна установить «Кыргызстан». Проверьте для других кодов.
  • Валидация обязательных полей: попробуйте сохранить кандидата, удалив ФИО, телефон или проект – должно появиться красное уведомление.
  • Выход: нажмите кнопку «Выйти» – должна появиться форма входа, сессия должна очиститься.
  • Восстановление сессии: после успешного входа обновите страницу – вы должны остаться в панели (сессия восстанавливается из sessionStorage).
  • Счётчик кандидатов: должен показывать «Кандидат X из Y». Если кандидатов нет – «Нет кандидатов для обработки».
