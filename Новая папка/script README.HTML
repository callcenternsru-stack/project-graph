📄 Инструкция: script.iss

Скрипт для компиляции установщика программы SystemCheckPRO с помощью Inno Setup.

Файл расположен: C:\Users\Контакт-Сервис\Desktop\prodect\PC_Checker\script.iss

2. Структура и семантика

Скрипт Inno Setup разделён на секции, каждая из которых начинается с имени в квадратных скобках. В данном файле присутствуют следующие секции:

  • Определения пользовательских констант (#define) – в начале файла задаются символические имена для названия приложения, версии, издателя, URL и имени исполняемого файла. Это облегчает поддержку скрипта.
  • [Setup] – обязательная секция, содержащая основные параметры установки:
    • AppId – уникальный идентификатор приложения (GUID).
    • AppName, AppVersion, AppPublisher, AppPublisherURL, AppSupportURL, AppUpdatesURL – метаданные приложения.
    • DefaultDirName – папка установки по умолчанию ({autopf} – папка Program Files).
    • UninstallDisplayIcon – иконка, отображаемая в списке установленных программ.
    • ArchitecturesAllowed и ArchitecturesInstallIn64BitMode – поддержка 64-битных систем.
    • DisableProgramGroupPage – отключает создание папки в меню "Пуск".
    • OutputDir и OutputBaseFilename – папка и имя выходного файла установщика.
    • SolidCompression – включает сжатие.
    • WizardStyle – стиль мастера установки.
    • SetupIconFile – путь к иконке для установщика.
    • PrivilegesRequired – требует права администратора для установки.
  • [Languages] – определяет языки мастера установки (здесь только английский).
  • [Tasks] – задаёт дополнительные задачи, которые пользователь может выбрать при установке. В данном случае – создание ярлыка на рабочем столе (по умолчанию выключено).
  • [Files] – самая объёмная секция, перечисляет все файлы, которые необходимо скопировать в папку установки. Каждая запись содержит исходный путь и путь назначения. Флаг ignoreversion означает, что при обновлении существующие файлы не будут проверяться на версию.
  • [Icons] – создаёт ярлыки в меню "Пуск" и (опционально) на рабочем столе.
  • [Run] – указывает, что после завершения установки можно запустить программу (опция с флагами nowait postinstall skipifsilent).

6. Особенности реализации

  • Пути к файлам: все исходные пути жёстко прописаны как абсолютные (C:\Users\Контакт-Сервис\Desktop\prodect\PC_Checker\...). Это делает скрипт немобильным – для сборки на другом компьютере потребуется либо точно такая же структура папок, либо изменение всех путей.
  • Иконка установщика: SetupIconFile указывает на favicon.ico. Если файл отсутствует, компиляция завершится ошибкой.
  • Права администратора: PrivilegesRequired=admin гарантирует, что установщик запросит повышение прав (UAC). Это необходимо, так как программа устанавливается в Program Files и требует прав для записи.
  • Архитектура: ArchitecturesInstallIn64BitMode=x64compatible означает, что установщик будет работать как на 32-битных, так и на 64-битных системах, но в 64-битном режиме будет использовать 64-битные пути (например, Program Files вместо Program Files (x86)).
  • Множество DLL: в секции [Files] перечислено множество DLL-библиотек, необходимых для работы sox.exe и, возможно, других утилит. Это гарантирует, что программа будет работать на любом компьютере без предустановленных зависимостей.
  • Дублирующиеся файлы: в списке есть report.txt, result.json, results.json, results.rar – это, вероятно, примеры или временные файлы. Некоторые из них могут быть лишними.
  • Ярлыки: создаётся ярлык в меню "Пуск" (всегда) и опционально на рабочем столе (если пользователь выбрал задачу).
  • Запуск после установки: секция [Run] предлагает пользователю запустить программу сразу после установки (галочка в мастере).

7. Потенциальные точки расширения

  • Относительные пути: заменить абсолютные пути на относительные, используя директиву Source: "PC_Checker\*" или разместив скрипт в корне проекта. Это сделает сборку переносимой.
  • Добавление лицензионного соглашения: можно включить файл лицензии (LicenseFile) и показывать его пользователю перед установкой.
  • Регистрация в реестре: при необходимости можно добавить записи в реестр (секция [Registry]), например, для ассоциации файлов.
  • Проверка наличия .NET Framework или других компонентов: через секцию [Run] с условиями.
  • Многоязычность: добавить русский язык в секцию [Languages] и соответствующие файлы сообщений.
  • Удаление старых версий: можно добавить проверку и удаление предыдущей установки (через [InstallDelete] или код на Pascal).
  • Цифровая подпись: добавить параметры подписи (SignTool) для подписывания установщика.
  • Сжатие: можно изменить метод сжатия (Compression) для оптимизации размера или скорости.

8. Особенности для тестирования

  • Компиляция скрипта: для компиляции требуется установленный Inno Setup (желательно последней версии). Откройте скрипт в среде Inno Setup и нажмите "Compile" (или выполните iscc script.iss из командной строки).
  • Проверка наличия всех файлов: перед компиляцией убедитесь, что все перечисленные файлы существуют по указанным путям. Отсутствие любого файла приведёт к ошибке компиляции.
  • Тестирование установки: запустите сгенерированный SystemCheckPRO_Setup.exe на чистой системе (или в виртуальной машине). Проверьте:
    • Запрос прав администратора (UAC).
    • Корректность предлагаемой папки установки.
    • Создание ярлыков в меню "Пуск" и на рабочем столе (если выбрано).
    • Запуск программы после установки (если отмечено).
    • Корректную работу программы (возможность ввести код и пройти тесты).
  • Тестирование удаления: после установки проверьте, что приложение можно удалить через "Панель управления" -> "Программы и компоненты". Убедитесь, что все файлы удалены.
  • Проверка целостности: после установки проверьте, что все необходимые DLL и утилиты находятся в папке установки (особенно sox.exe, speedtest.exe).
  • Логирование: при возникновении ошибок установки можно включить логирование Inno Setup, добавив параметр /LOG при запуске установщика.
  • Размер установщика: проверьте, что итоговый файл не слишком велик (если да, можно исключить ненужные файлы, например, дублирующиеся results.*).
