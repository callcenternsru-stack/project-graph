2. Структура и семантика

Страница состоит из простых семантических блоков:

  • Шапка (.header) – содержит заголовок «Графики работы» и панель управления с двумя выпадающими списками: выбор проекта (#projectSelect) и выбор месяца (#monthSelect).
  • Контейнер графика (#scheduleContainer) – изначально отображает индикатор загрузки (.loader). После загрузки данных здесь строится таблица графика (.schedule-wrapper).
  • Таблица графика – реализована с помощью flex и состоит из:
    • фиксированной левой части (.fixed-headers и .fixed-cells) с колонками: ФИО, Смена, Время.
    • прокручиваемой правой части (.scrollable-headers и .scrollable-cells) с ячейками для каждого дня месяца.

Вся вёрстка рассчитана на широкие экраны и горизонтальный скролл при необходимости. Сообщения об ошибках или отсутствии данных отображаются в том же контейнере.

3. Стилизация (CSS)

Корпоративная тёмная тема: фон #000 и #111, акцентный цвет #ffd700 (золотой/жёлтый), текст белый/серый.

  • Шапка – тёмная карточка с рамкой, заголовок жёлтый. Элементы управления (select) стилизованы под тёмную тему, при фокусе получают жёлтую рамку.
  • Таблица графика:
    • Фиксированные заголовки и ячейки имеют жёлтый текст, фон #000 или #111.
    • Ячейки времени (.time-value) имеют тёмный фон, рамку, скругление. При наличии значения добавляется класс filled (не используется в JS, но есть в CSS).
    • Статусы ячеек окрашиваются через классы: .late (жёлтый фон), .overtime (зелёный), .absent (красный), .sick (синий), .vacation (синий), .dayoff (серый).
  • Индикатор загрузки (.loader) – жёлтый текст по центру.
  • Сообщения об ошибках (.error-message) – красный текст.
  • Сообщение об отсутствии данных (.no-data) – серый текст.

Стили полностью самодостаточны и не используют внешние библиотеки.

4. JavaScript (логика и интерактивность)

Код написан на чистом JavaScript (ES6) и включает следующие функции:

  • Глобальные переменные:
    • GET_DATA_URL – адрес функции getData.
    • allData – объект, содержащий полные данные, загруженные с сервера.
  • Загрузка данных:
    • loadData() – асинхронно загружает данные через fetch, проверяет успешность, сохраняет в allData и вызывает fillProjectSelect(). При ошибке показывает сообщение через showError().
    • showError(msg) – заменяет содержимое контейнера графика на сообщение об ошибке.
  • Заполнение селектов:
    • fillProjectSelect() – заполняет #projectSelect списком проектов из allData.projects. Если проектов нет, показывает соответствующую опцию. Затем вызывает updateMonthSelect(true) для загрузки месяцев и отображения графика.
    • updateMonthSelect(shouldLoad) – по выбранному проекту получает массив месяцев из allData.monthsByProject, сортирует по убыванию и заполняет #monthSelect отформатированными названиями (используя formatMonth()). Если shouldLoad истинно, вызывает loadSchedule().
  • Вспомогательные функции для дат и времени:
    • formatMonth(monthStr) – преобразует строку ГГГГ-ММ в название месяца и год (например, «Март 2026»).
    • getDaysInMonth(monthStr) – возвращает количество дней в указанном месяце.
    • getDayOfWeek(monthStr, day) – возвращает сокращённое название дня недели (вс … сб).
    • getDuration(t) – внутренняя функция, вычисляет продолжительность временного интервала в минутах из строки вида ЧЧ:ММ-ЧЧ:ММ.
    • getCellStatus(value, baseTime) – определяет статус ячейки на основе значения и базового времени. Возвращает класс: late, overtime, absent, sick, vacation, dayoff или пустую строку.
    • cleanEmployeeName(fullName) – удаляет суффикс (партнёр) из имени сотрудника для отображения чистого ФИО.
  • Построение графика:
    • loadSchedule() – основная функция рендеринга. Получает выбранные проект и месяц, проверяет наличие данных. Если есть сотрудники для указанного периода, строит HTML-таблицу с фиксированными колонками и прокручиваемыми днями. Для каждого сотрудника извлекает базовые данные (смена, время) и для каждого дня – значение из allData.entries, вычисляет статус и применяет соответствующий класс. После вставки HTML корректирует ширину фиксированных колонок, чтобы они совпадали с шириной заголовков.
  • Обработчики событий:
    • document.getElementById('projectSelect').addEventListener('change', ...) – при смене проекта вызывает updateMonthSelect(true).
    • document.getElementById('monthSelect').addEventListener('change', loadSchedule) – при смене месяца перерисовывает график.
    • window.addEventListener('load', loadData) – при загрузке страницы инициирует получение данных.

5. Форма и отправка данных

На странице нет форм для отправки данных. Она только загружает и отображает информацию. Единственное взаимодействие с сервером – GET-запрос к /.netlify/functions/getData. Никакие изменения не сохраняются.

6. Особенности реализации

  • Только чтение: страница предназначена для публичного доступа, поэтому не содержит элементов редактирования (нет input'ов, кнопок сохранения и т.д.).
  • Синхронизация с админкой: использует те же структуры данных (projects, monthsByProject, employees, employeeData, entries), что и index.html и partner.html. Это гарантирует, что любые изменения, сделанные в админке, сразу видны на публичной странице (после перезагрузки).
  • Обработка имён сотрудников с суффиксом партнёра: функция cleanEmployeeName() отрезает часть (Имя партнёра), чтобы показывать только ФИО оператора. Это делает отображение более чистым.
  • Сортировка месяцев: месяцы отображаются в обратном хронологическом порядке (сначала новые), что удобно для просмотра актуальных данных.
  • Адаптация ширины фиксированных колонок: после рендеринга ширина блока .fixed-cells устанавливается равной ширине блока .fixed-headers, чтобы левая часть таблицы не съезжала при скролле.
  • Цветовая индикация: статусы ячеек вычисляются по тем же правилам, что и в панели партнёра, что обеспечивает единообразие.
  • Обработка ошибок: при неудачной загрузке данных или отсутствии проектов/месяцев/сотрудников показываются понятные сообщения.

7. Потенциальные точки расширения

  • Фильтр по сотрудникам: добавить поле поиска или выпадающий список для выбора конкретного сотрудника (сейчас отображаются все).
  • Экспорт в Excel/PDF: добавить кнопку для выгрузки текущего графика в формате CSV или Excel.
  • Легенда цветов: добавить пояснение, что означают цвета ячеек (как на странице партнёра).
  • Кэширование данных: можно сохранять загруженные данные в localStorage и показывать их при повторном посещении, фоном обновляя.
  • Периодическое обновление: автоматически перезагружать данные каждые несколько минут, чтобы отображать актуальную информацию без перезагрузки страницы.
  • Параметры URL: добавить возможность передавать проект и месяц в query-параметрах, чтобы можно было давать прямую ссылку на конкретный график.
  • Мобильная версия: улучшить адаптивность для маленьких экранов (например, сворачивать фиксированные колонки в выпадающий список).
  • Подсветка выходных: добавить визуальное выделение суббот и воскресений.

8. Особенности для тестирования

  • Необходимость сервера: страница требует работающей Netlify Function getData. При локальном открытии file:// запрос не сможет выполниться. Для тестирования используйте развёрнутый сайт или Netlify CLI.
  • Проверка загрузки: при открытии страницы должен появиться индикатор загрузки, затем – селекты проектов и месяцев. Если данные не загружаются, отобразится сообщение об ошибке.
  • Выбор проекта и месяца: убедитесь, что при смене проекта список месяцев обновляется, а при смене месяца – перерисовывается график.
  • Отображение данных: проверьте, что для выбранного периода отображаются сотрудники, их смены, базовое время и значения по дням. Ячейки должны иметь правильные цвета в зависимости от статуса.
  • Горизонтальный скролл: если дней в месяце много, должна появиться горизонтальная полоса прокрутки. Фиксированные колонки (ФИО, смена, время) должны оставаться на месте при скролле.
  • Пустые состояния: проверьте проект, в котором нет месяцев, или месяц, в котором нет сотрудников – должно отобразиться соответствующее сообщение.
  • Корректность имён: убедитесь, что имена сотрудников отображаются без суффикса партнёра (например, «Иванов Иван» вместо «Иванов Иван (ООО Партнёр)»).
  • Цветовая схема: при вводе тестовых значений (например, «прг», «б», времени меньше/больше базового) ячейки должны окрашиваться в соответствующие цвета.
  • Консоль браузера: при выполнении всех операций не должно быть ошибок в консоли.
