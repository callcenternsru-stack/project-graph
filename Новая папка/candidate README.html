2. Структура и семантика

Страница разделена на логические блоки, оформленные с помощью семантических тегов <section>, <header> (неявно через заголовки), списков и форм.

  • Шапка (h1) – название проекта «РОССЕТТИ · Анкета кандидата».
  • Информация о проекте – описание, требования, условия (выделены в карточки .info-card).
  • FAQ (выпадающие блоки) – реализованы как аккордеон с помощью CSS и JavaScript (классы .faq-item, .faq-question, .faq-answer).
  • Сравнение способов проверки – ещё один выпадающий блок, объясняющий разницу между автоматической и ручной проверкой.
  • Форма анкеты (<form id="candidateForm">) – содержит обязательные поля: ФИО, псевдоним, Telegram, телефон, email. Также скрытые поля для хранения projectId и названия проекта.
  • Кнопки выбора способа проверки – две большие кнопки, запускающие соответствующие сценарии.
  • Блоки с результатами (.result-block) – скрыты по умолчанию, появляются после отправки формы. Содержат код для авто-проверки и инструкции, а для ручной – пошаговый интерфейс из 7 заданий.

Вся вёрстка адаптивная: используется grid для карточек и полей формы, медиазапрос для узких экранов (колонки складываются в одну).

3. Стилизация (CSS)

Корпоративная тёмная тема: фон #000 и #111, акцентный цвет #ffd700 (золотой/жёлтый), текст белый/серый.

  • Карточки .info-card – тёмный фон, скругления, тонкая рамка.
  • FAQ – анимация при открытии/закрытии через max-height и transition.
  • Форма – все поля ввода имеют тёмный фон, жёлтую обводку при фокусе.
  • Кнопки – две основные: жёлтая (первичная) и тёмная с жёлтой рамкой (вторичная). При наведении поднимаются и увеличивается тень.
  • Блоки результата – появляются с анимацией fadeIn, код отображается крупным моноширинным шрифтом.
  • Уведомления (.notification) – фиксированное всплывающее окно в правом нижнем углу, исчезает через 3 секунды.
  • Стили для ручной проверки – добавлены прогресс-бар, карточки заданий, стилизованные ссылки-кнопки, загрузка файлов, навигационные кнопки, а также стили для встроенного теста с вариантами ответов.
  • Индикатор загрузки – полноэкранный оверлей с анимированным спиннером и текстом «Идет загрузка...».

Все стили находятся внутри тега <style> и не используют внешние библиотеки.

4. JavaScript (логика и интерактивность)

Код написан на чистом JavaScript (ES6) и включает следующие функции:

  • getQueryParam() – чтение параметра projectId из URL.
  • loadProjects() – загрузка списка проектов с сервера и установка скрытых полей.
  • generateUniqueCode() – генерирует код формата ROSSETI-XXXX-XXXXXXXX.
  • validateForm() – проверяет заполненность обязательных полей формы.
  • getFormData() – собирает данные формы.
  • showNotification() – показывает временное уведомление.
  • showLoading() / hideLoading() – управление оверлеем загрузки.
  • Автоматическая проверка – startAutoVerification():
    • Валидирует форму, проверяет projectId.
    • Показывает оверлей загрузки.
    • Генерирует код, отправляет данные на submitForm.
    • По окончании скрывает загрузку и показывает блок с кодом.
  • Ручная проверка – полностью переработана:
    • Массив tasks содержит 7 заданий с описаниями, ссылками, требованиями к загрузке файлов и вопросами для теста.
    • renderTask() – отрисовывает текущее задание, включая прогресс, инструкции, ссылки (в т.ч. отдельную кнопку для примера AnyDesk), вопросы с radio-кнопками, поле ввода и загрузку файлов.
    • handleFileSelect(), handleInputChange(), handleQuizAnswer() – обработчики изменений.
    • updateButtons() – проверяет возможность перехода к следующему заданию (все обязательные поля заполнены) и активирует/деактивирует кнопки «Назад»/«Далее».
    • prevTask() / nextTask() – навигация по заданиям.
    • calculateQuizScore() – подсчёт процента правильных ответов для теста задания №3.
    • submitManualResults() – собирает все данные (включая файлы, ответы и результаты теста) в FormData и отправляет на submitManualResults, показывая оверлей загрузки.
  • copyCode() – копирует код в буфер обмена.
  • toggleFaq() – открывает/закрывает пункты FAQ.

5. Форма и отправка данных

Поля формы:

  • fullName (обязательное) – текстовое поле.
  • nickname (обязательное) – русскоязычный псевдоним.
  • telegram (обязательное) – логин Telegram (ожидается с @).
  • phone (обязательное) – номер телефона.
  • email (обязательное) – email.
  • projectId (скрытое) – уникальный ключ проекта, полученный из URL.
  • projectName (скрытое) – название проекта для отображения.

Валидация: проверка на пустое значение.

Отправка:

  • Автоматическая: POST на /submitForm с JSON { code, formData }.
  • Ручная: POST на /submitManualResults с multipart/form-data, включая файлы заданий и ответы.

Во время отправки показывается оверлей «Идет загрузка...», блокирующий повторные нажатия.

6. Особенности реализации

  • Передача projectId через URL: кандидат не видит и не может изменить проект, он определяется автоматически по параметру в ссылке, которую предоставил администратор.
  • Пошаговый интерфейс ручной проверки: задания открываются последовательно; кнопка «Далее» активируется только после выполнения требований текущего задания (загрузка файла, ввод номера, ответы на все вопросы).
  • Встроенный тест с вариантами ответов: вопросы отображаются с radio-кнопками, правильные ответы заданы в коде; после завершения теста подсчитывается процент правильных ответов и отправляется на сервер вместе с ответами.
  • Кнопка для примера AnyDesk: отображается как отдельная стилизованная ссылка рядом с основной.
  • Индикатор загрузки: полноэкранный оверлей с анимацией, появляется при отправке любой формы и при старте ручной проверки (для плавного перехода).
  • Защита от дубликатов: серверные функции (submitForm, submitManualResults) при сохранении удаляют предыдущие анкеты с теми же данными.

7. Потенциальные точки расширения

  • Динамическое добавление проектов: сейчас projectId жёстко зашит в ссылке; можно сделать выбор проекта из выпадающего списка, но текущая схема скрытой передачи удобнее.
  • Редактирование текстов заданий через админку: можно вынести описания и ссылки в JSON на сервере, чтобы HR мог менять их без изменения кода.
  • Более строгая валидация: добавить проверку формата email, телефона, допустимых расширений файлов.
  • Отслеживание статуса: после отправки авто-анкеты можно периодически опрашивать сервер и показывать результат прямо на странице.
  • Прогресс сохранения: индикатор загрузки можно дополнить процентом отправки файлов.

8. Особенности для тестирования

  • Необходимость сервера: для полноценного тестирования требуется работающий бэкенд (Netlify Functions).
  • Тестовые данные: используйте ссылку вида https://grafic1.netlify.app/candidate.html?projectId=PROJ-XXXX-XXXX (подставьте реальный ID).
  • Проверка валидации: оставьте поля пустыми – должны подсветиться красным.
  • Ручная проверка: пройдите все 7 заданий, убедитесь, что кнопка «Далее» активируется только после выполнения условий, а на последнем шаге становится «Отправить».
  • Тест с вариантами: выберите ответы на все вопросы – только тогда станет доступен переход. После отправки проверьте в HR-панели полученные данные.
  • Индикатор загрузки: при нажатии «Отправить» должен появиться оверлей и исчезнуть после ответа сервера.
  • Консоль браузера: не должно быть ошибок.

9. Новое в версии (март 2026): черновик ручной проверки, отказ от уведомлений и клиентское сжатие файлов

  • Мгновенное сохранение черновика: при нажатии кнопки «Пройти проверку вручную» данные анкеты сразу отправляются на сервер и создаётся запись в хранилище manualForms со статусом draft. Благодаря этому анкета сразу появляется в HR-панели, даже если кандидат ещё не завершил все задания.
  • Обновление при завершении: после выполнения всех семи заданий и нажатия «Отправить» та же запись обновляется: к ней добавляются загруженные файлы, ответы на тесты, результаты, а статус меняется на completed. Таким образом, в HR-панели сохраняется полная история.
  • Глобальная переменная currentManualId: в JavaScript добавлена переменная, хранящая ID созданного черновика. Этот ID передаётся при финальной отправке, чтобы сервер знал, какую запись обновить.
  • Модификация серверной функции submitManualResults: теперь она умеет как создавать новые записи (без поля id), так и обновлять существующие (если поле id передано). Файлы сохраняются в подпапке с именем ID записи.
  • Отказ от уведомлений об успехе: для улучшения пользовательского опыта убраны всплывающие уведомления об успешной отправке анкеты (как автоматической, так и ручной). Теперь уведомления показываются только в случае ошибки. Уведомление о копировании кода также удалено – остался только alert при неудаче.
  • Клиентское сжатие файлов: для соблюдения лимитов Netlify (10 МБ) добавлено сжатие изображений (до 1 МБ, 1280px) и конвертация WAV-аудио в MP3 (96 kbps) непосредственно в браузере перед отправкой. Это позволяет загружать до 7 файлов общим размером менее 10 МБ. При ошибке конвертации аудио отправляется оригинальный файл – данные не теряются.
  • Преимущества: HR видит всех начавших проверку кандидатов, может отслеживать прогресс и при необходимости связываться с ними. Исключена ситуация, когда кандидат заполнил форму, но не завершил задания, и анкета потерялась. Файлы передаются стабильно, без ошибок 502.
