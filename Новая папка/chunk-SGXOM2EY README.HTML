2. Структура и семантика

Файл представляет собой набор независимых утилит, объединенных в один модуль для внутреннего использования библиотекой Netlify Blobs:

  • Заголовки: определяет константы для заголовков ошибок (NF_ERROR, NF_REQUEST_ID).
  • Утилиты:
    • BlobsInternalError — формирует детализированное сообщение об ошибке из заголовков ответа.
    • collectIterator — удобная функция для сбора всех элементов асинхронного итератора.
    • isNodeError — type guard для ошибок Node.js.
    • base64Decode / base64Encode — кроссплатформенная реализация (использует Buffer в Node.js, atob/btoa в браузере).
  • Окружение:
    • getEnvironment — адаптер для доступа к переменным окружения в разных средах (Node.js, Deno, Netlify).
    • getEnvironmentContext / setEnvironmentContext — управление глобальным контекстом (сериализуется в base64 и хранится в NETLIFY_BLOBS_CONTEXT).
    • MissingBlobsEnvironmentError — ошибка, выбрасываемая при отсутствии обязательных параметров.
  • Метаданные:
    • METADATA_HEADER_INTERNAL (x-amz-meta-user) и METADATA_HEADER_EXTERNAL (netlify-blobs-metadata) — заголовки для передачи метаданных.
    • encodeMetadata — преобразует объект метаданных в строку формата b64;base64, проверяет размер (не более 2 КБ).
    • decodeMetadata — обратное преобразование.
    • getMetadataFromResponse — извлекает метаданные из ответа (сначала ищет внешний заголовок, затем внутренний).
  • Согласованность: BlobsConsistencyError — ошибка при запросе strong consistency без настроенного uncachedEdgeURL.
  • Повторные попытки:
    • fetchAndRetry — обертка вокруг fetch с повторными попытками при ошибках 429, 500+ или сетевых сбоях.
    • getDelay — вычисление задержки с учетом заголовка X-RateLimit-Reset.
    • sleep — простая задержка на Promise.
    • MAX_RETRY = 5, DEFAULT_RETRY_DELAY = 5000 мс (или 1 мс в тестовом режиме).
  • HTTP-клиент:
    • Класс Client — основной клиент для взаимодействия с API.
    • Поддерживает два режима: через Edge URL (быстрый, кэширующий) и через API URL (прямой).
    • Генерирует подписанные URL для загрузки/скачивания.
    • Управляет заголовками авторизации и метаданными.
    • Обрабатывает consistency (eventual/strong).
    • Поддерживает регионы через параметр region или префикс /region:{region} в пути.
    • getClientOptions — фабрика для создания опций клиента, объединяющая параметры из контекста и явно переданные значения.

3. Стилизация (CSS) — не применимо

Данный файл является серверным/клиентским JavaScript-модулем и не содержит элементов пользовательского интерфейса.

4. JavaScript (логика и интерактивность)

  • Кроссплатформенность: все функции учитывают различные окружения (Node.js, Deno, браузер) и используют соответствующие глобальные объекты.
  • Потоки: поддержка ReadableStream в теле запроса с установкой duplex: "half" для корректной работы в браузерах.
  • Повторные попытки: реализованы с экспоненциальной задержкой и учетом rate limit.
  • Обработка ошибок: специфические классы ошибок для разных ситуаций (внутренние, консистентность, отсутствие конфигурации).
  • Метаданные: кодируются в base64 с префиксом для безопасной передачи в заголовках.
  • Контекст окружения: глобальный объект globalThis.netlifyBlobsContext или переменная NETLIFY_BLOBS_CONTEXT хранят сериализованный контекст.
  • Подписанные URL: клиент запрашивает подписанный URL для операций с данными, что повышает безопасность.
  • Валидация: проверка наличия обязательных параметров (siteID, token) и выбрасывание MissingBlobsEnvironmentError.

5. Форма и отправка данных

Модуль не содержит пользовательских форм. Он обеспечивает транспортный уровень для HTTP-запросов:

  • Формирование URL: в зависимости от режима (Edge/API) и параметров (регион, store, key).
  • Заголовки: авторизация (Bearer token), метаданные, cache-control, accept.
  • Тело запроса: может быть строкой, Buffer, Blob, ReadableStream.
  • Ответы: обрабатываются с учетом статусов и заголовков.

6. Особенности реализации

  • Адаптивность окружения: через getEnvironment модуль работает в Node.js, Deno, браузерах и Netlify Edge.
  • Безопасность: подписанные URL исключают необходимость передавать токен в каждом запросе.
  • Консистентность: поддержка eventual и strong consistency через разные URL (cached/uncached).
  • Регионы: поддержка регионов через параметр или префикс пути (для Edge).
  • Rate limit handling: автоматические повторные попытки с учетом заголовка X-RateLimit-Reset.
  • Размер метаданных: строгое ограничение в 2 КБ для предотвращения проблем с заголовками.
  • Глобальный контекст: позволяет передавать конфигурацию между модулями без явных параметров.
  • Тестирование: в режиме тестирования (NODE_ENV=test) задержка повторных попыток сокращается до 1 мс для ускорения тестов.
  • Совместимость: используется везде, где есть fetch, включая Cloudflare Workers, Deno, современный Node.js.

7. Потенциальные точки расширения

  • Поддержка разных алгоритмов повторных попыток: добавить экспоненциальную задержку с jitter.
  • Сжатие метаданных: для уменьшения размера при больших объектах.
  • Кэширование подписанных URL: чтобы не запрашивать их повторно для одних и тех же операций.
  • Мониторинг: добавить сбор метрик по количеству запросов, повторным попыткам, ошибкам.
  • Поддержка HTTP/2: для улучшения производительности.
  • Гибкая настройка таймаутов: возможность задать timeout для запросов.
  • Логирование: встроенное логирование запросов для отладки.

8. Особенности для тестирования

  • Мокирование fetch: для тестирования можно подменить globalThis.fetch или передать свою реализацию в конструктор Client.
  • Тестирование повторных попыток: установить NODE_ENV=test для сокращения задержек.
  • Проверка кодирования метаданных: 
    const meta = { user: 'john' };
const encoded = encodeMetadata(meta);
const decoded = decodeMetadata(encoded);
assert.deepEqual(decoded, meta);
  • Тестирование ошибок: проверить, что BlobsInternalError корректно извлекает заголовки.
  • Тестирование контекста: 
    setEnvironmentContext({ siteID: 'test', token: '123' });
const ctx = getEnvironmentContext();
assert.equal(ctx.siteID, 'test');
  • Тестирование клиента с локальным сервером: запустить BlobsServer и передать его URL как edgeURL.
  • Тестирование strong consistency: передать uncachedEdgeURL и проверить, что запросы идут на него.
  • Тестирование rate limit: замокать ответ с заголовком X-RateLimit-Reset и проверить, что повторные попытки происходят с правильной задержкой.
  • Кросс-средовое тестирование: проверить работу в Node.js, браузере (с atob/btoa), Deno.
